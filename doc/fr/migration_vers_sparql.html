Ce document est une conclusion d'une étude sur le SI d'une entreprise, en vue d'une migration vers un serveur SPARQL qui unifie les données.

Les grands types de données

Le Système d'Information (SI) documentaire de XXX présente une certaine complexité due à la présence de nombreuses applications spécialisées, dont les fonctionnalités se recouvrent, qui constituent autant de silos de données cloisonnés, et qui nécessitent un nombre important de flux manuels. On a par exemple plusieurs CMS . On a plusieurs outils pour saisir des données structurées par formulaires . Pour les données de type thésaurus, on a principalement un outil, mais des thésaurus déconnectés, ou connectés une fois par recopie de données, existent ailleurs.

Tous ces types de données peuvent s'intégrer et être mises à jour dans une base au standard SPARQL du Web Sémantique.

données structurées par propriété, valeur, et classe

thésaurus : données de type mots-clé, hiérarchisées

textes plus ou moins structurés (formats HTML, InDesign)

Possibilités du futur Entrepôt Sémantique

En interne le Web Sémantique est le moyen idéal de réaliser une intégration des données en temps réel, en décloisonnant les données, les modèles et les créateurs de la donnée. Le tout en préservant là où c'est utile une diversité de points de vues.

Le méta modèle de données du Web Sémantique est ouvert sur l'Intelligence Artificielle, car il favorise une sémantique unifiée et rigoureuse. Il inclut la notion d'inférence logique.

L'EDD (Entrepôt De Données) permettra facilitera toutes sortes d'études permettant de valoriser les données. C'est à dire tout ce qui rentre sous les appellations BI, Informatique décisionnelle, big data.

Le point de départ des travaux BI est toujours une phase de modélisation globale. Avec les technologies sémantiques, ce sera déjà fait. Et avec la puissance des bases SPARQL, on peut à la fois répondre aux besoins de production et de BI, c'est à dire qu'on fait d'une pierre deux coups.

Le décloisonnement permettra de décloisonner, croiser et recouper les données de manières créatives et non nécessairement prévues au départ. Dans cet ordre d'idées, le nettoyage des données sera facilité, à l'aide d'assistants logiciels branchés directement sur l'EDD.

En fait, on pourra disposer d'outils qui permettront d'éditer directement les données de l'EDD, et différents outils d'administration. Dans le vaste écosystème du Web Sémantique, on trouve des composants d'Interface Homme Machine et de formulaires qui peuvent se brancher sur des données sémantiques, chercher sur Internet : semantic web forms, notamment :

  • projet: semantic_forms, exemples : example.form.foaf.html , display FOAF

  • projet: viejs.org, exemple : http://viejs.org/widgets/forms/

L'EDD pourrait centraliser les micro-données en vue du SEO à répercuter dans les sites Web de l'entreprise.

Vis à vis de l'extérieur, un entrepôt SPARQL fournira une infrastructure commode pour être un référent sur les données publiques maîtrisées par l'entreprise, en France d'abord, mais pourquoi pas ailleurs …En effet, fournir et maintenir des URI pour ces données rend évident pour les acteurs du secteur que c'est la référence.

Il faudrait peut-être, réfléchir à une licence compatible avec les données ouvertes, mais préservant les prérogatives de propriété intellectuelle.

Il faudrait envisager d'utiliser davantage les données ouvertes (Linked Open Data), comme. Wikipedia et Wikidata. Embrasser l'écosystème mondial du Web Sémantique est une chance historique pour l'entreprise.

L' Infrastructure de l'EDD, par l'unification des données, facilitera des retours d'informations ciblés vers les parties prenantes et acteurs de la vie économique, culturelle et scientifique.
